## 🔍 증거 분석

### A. 구조적 손상 패턴

#### 증거 1: 함수 정의 들여쓰기 오류의 일관성

**패턴**: 함수의 **첫 번째 코드 라인이 항상** 들여쓰기되지 않음

```
Lesson 2:
def show_kernel(...):
    # 주석
    kernel = np.array(...)
# ❌ 여기서 들여쓰기 끝남

Lesson 4:
def circle(...):
    circle = np.zeros(...)
rr, cc = draw.circle_perimeter(...)  # ❌ 들여쓰기 없음
```

**자동 번역 도구의 증거**:
- 마크다운 → HTML → 마크다운 변환 과정에서 코드 블록의 들여쓰기가 손실될 가능성
- 구조화된 텍스트를 재구성하지 않고 라인 단위로만 번역하면 발생

#### 증거 2: 공백 오류의 체계성

**패턴**: `.` 연산자 앞뒤에 공백이 추가됨

| 오류 | 발생 위치 |
|------|---------|
| `tf.keras. Sequential` | Lesson 4, Line 63 |
| `tf.keras.layers. Activation` | Lesson 4, Line 72 |
| `np.random.seed (seed)` | Lesson 1, Line 83 |
| `.map (convert_to_float)` | Lesson 5, 6, Lines 87, 91 |

**분석**:
- `.` 앞뒤의 공백은 **수동 오류가 아님** (일관성이 있음)
- 자동 번역 → OCR 또는 정규식 기반 처리 → 공백 추가의 가능성

#### 증거 3: 수식 파괴 (Lesson 4만)

**패턴**: 한자리 또는 두자리 수식이 여러 라인에 걸쳐 분해됨

```
원본: 2×2
❌ 변환됨:
2
×
2
```

**분석**:
- 이것은 **자동 번역 도구의 줄바꿈 처리 오류**
- 수동 번역자는 절대 이런 실수를 하지 않음

## 🤖 자동 번역 도구 추적

### 가능한 도구들

| 도구 | 가능성 | 증거 |
|------|--------|------|
| **Google Translate** | ⭐⭐⭐⭐⭐ 매우 높음 | 들여쓰기 손상, 공백 오류 패턴 일치 |
| **대충한 ChatGPT** | ⭐⭐⭐⭐ 높음 | HTML 변환 오류 가능성 |
| **Azure Translator** | ⭐⭐⭐ 중간 | 특정 패턴과 일부 일치 |
| **파파고** | ⭐⭐ 낮음 | 한국어 기술 문서 상대적으로 우수 |

### 가장 가능성 높은 시나리오

```
1. 원본 영어 마크다운
           ↓
2. HTML로 변환 (웹 렌더링)
           ↓
3. Google Translate (또는 유사 도구)로 번역
           ↓
4. HTML → 마크다운 변환 (Markdownify 등)
           ↓
5. 결과: 구조 손상 + 공백 오류
```

**이 과정의 문제점**:
- 단계 2에서 들여쓰기 정보 일부 손실
- 단계 4에서 부분적으로만 복구
- 결과적으로 특정 라인의 들여쓰기만 손상

## 💡 근본 원인 분석

### 1. 마크다운 → HTML 변환 오류

**문제**: 마크다운의 들여쓰기가 HTML 렌더링 시 무시되거나 변환될 수 있음

```markdown
# 원본 마크다운
def show_kernel(...):
    kernel = np.array(...)
    if digits is not None:
        kernel = kernel.round(digits)
```

→ HTML 변환 시 `<pre>` 태그 처리에 따라 들여쓰기가 손실될 수 있음

### 2. 자동 번역 도구의 한계

**문제**: 일반 번역 도구는 코드 블록의 구조를 이해하지 못함

```python
# 번역 도구가 보는 것:
"def show_kernel ... kernel = np.array ... if digits is not None ..."
# (한 줄의 텍스트로 취급)

# 번역 후 마크다운으로 변환할 때:
들여쓰기 정보를 복구할 수 없음 ❌
```

### 3. Markdownify 등 역변환 도구의 한계

**문제**: HTML → 마크다운 변환 시 코드 블록의 원본 들여쓰기를 완벽히 복구할 수 없음

```
원본 마크다운 
→ HTML 렌더링 
→ 자동 번역
```

✅ **올바른 방식**:
```
원본 마크다운 
→ 텍스트만 추출 (코드 블록은 제외!)
→ 자동 번역 (텍스트만)
→ 코드 블록 수동 검증/번역
→ 최종 마크다운 조립
```

또는:

✅ **더 좋은 방식**:
```
원본 마크다운 읽음
→ 코드 블록 표시하고 제외
→ 텍스트만 번역
→ 코드 블록 다시 삽입
→ 수동 검증
```

## 📋 결론

### 현재 상태
- **6개 중 3개** 레슨이 우수한 수동 번역으로 완성
- **6개 중 3개** 레슨이 자동 번역으로 인한 구조적 손상 존재

### 원인

- **마크다운 변환 오류**: HTML → 마크다운 변환 과정에서 정보 손실
- **검증 부재**: 번역 후 코드 블록 검증이 이루어지지 않음

### 해결책
1. **즉시**: 우선순위에 따라 오류 수정
2. **단기**: 이번 수정과 동시에 예방 프로세스 구축
3. **장기**: 자동화된 품질 관리 시스템 도입

- 수정 후 평균 평점: 3.7/5.0 → **4.5/5.0 이상**
